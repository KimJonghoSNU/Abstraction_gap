## 2026-02-04 추가: `round3_anchor_local_rank` v5/v6 정리
- 공통
    - 평가/컨텍스트의 anchor 순서는 branch 치환(v2) 기반이 아니라 `anchor_local_rank=none`과 동일한 leaf-only anchor 순서를 seed로 사용.
    - seed 10 + expanded 10 = candidate 20을 만든 뒤, LLM rerank로 최종 top-10을 선택.
- `v5`
    - candidate 20을 leaf-only anchor 상위 leaf들로 구성.
    - 그 20개를 LLM이 relevance 기준으로 재정렬/필터링하고 top-10을 확정.
- `v6`
    - 먼저 leaf-only anchor top-10(seed)을 만든다.
    - extra 10은 query 점수가 아니라, 각 candidate leaf에 대해
      `max_{seed in top10} cosine(emb(candidate), emb(seed))` 로 점수화해 선별.
    - 추가 제약: seed의 `doc.id prefix`( `.txt`, trailing `_<숫자>` 제거)와 같은 prefix의 candidate는 제외.
    - 즉, v6는 "현재 top-10과 임베딩 공간에서 가까운 이웃"을 모아서 rerank하는 PRF-like 확장이다.

## 2026-02-06 업데이트: Category-focused explore/exploit (soft policy)
- 목표: explore/exploit을 **category 선택 정책**으로 연결.
    - exploit: evidence로 더 지지되는 category에 query를 집중.
    - explore: 이전 category를 제외하지 않고 포함한 채로 재평가하되, `beta` 패널티로 과도한 재사용을 완화.
- 핵심 아이디어:
    - `round3_action_v1`처럼 global action만 있어도 `Possible_Answer_Docs`를 category별로 파싱해 정책을 적용.
    - query 구성 시 선택된 category만 남기고 나머지는 PRUNE 처리.
    - `soft_keep=1`이면 hard 집중과 동일.
- 구현 (run_round3_1.py):
    - `Possible_Answer_Docs`를 저장하고(`_parse_action_output`), category selector 입력으로 사용.
    - category support 점수:
        - 입력 evidence: 현재 iter anchor leaf top-`round3_category_support_topk` (default 10).
        - 각 category text와 evidence text 임베딩 cosine 평균(top-m)을 support로 계산.
    - explore penalty:
        - `final = support - beta * similarity_to_prev_selected_category`
        - 여기서 previous category는 **제외하지 않음**. 점수 경쟁으로 top1/topK 선택.
    - 선택된 category만 `query_t` 구성에 사용 (`_compose_query_from_docs_with_policy`).
    - 로그:
        - `query_categories` (실제 retrieval query에 반영된 category)
        - `category_scores`, `category_support_scores`, `category_penalty_scores`
- Iteration X 알고리즘 요약:
    - 1) Iter X 시작 시, Iter X-1에서 선택된 category들로 `query_t`를 구성해 anchor retrieval 수행.
    - 2) rewrite 조건이 만족되면 LLM(`round3_action_v1`) 호출 → `action` + `Possible_Answer_Docs` 생성.
    - 3) category selector가 evidence top-10과 category docs의 cosine support를 계산하고,
         explore면 `beta` 패널티를 적용해 top-`soft_keep` category를 선택.
    - 4) 선택 category만 남긴 rewrite를 저장.
    - 5) Iter X+1 retrieval은 이 선택 결과로 다시 시작.
- 하이퍼파라미터:
    - `--round3_category_policy {none,soft}` (default: none)
    - `--round3_category_soft_keep N` (default: 2, N=1이면 hard)
    - `--round3_category_support_topk` (default: 10)
    - `--round3_category_explore_beta` (default: 0.1)
- 사용 예시:
    - Soft: `--round3_category_policy soft --round3_category_soft_keep 2`
    - Hard: `--round3_category_policy soft --round3_category_soft_keep 1`
