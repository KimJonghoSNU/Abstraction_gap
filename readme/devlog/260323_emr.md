# 2026-03-23 EMR-Style Memory for Baseline3 Leaf Loop

## 목적
- Tree 없이 동작하는 `baseline3_leaf_only_loop`에 EMR-style memory를 붙여, iterative query refinement에서 memory 자체의 효과를 분리해서 보려는 목적.
- Retrieval / ranking / scoring은 그대로 두고, rewrite prompt에만 memory를 추가하는 prompt-only ablation으로 구현.
- 현재 paper-faithful main setting은 `accumulated`다.

## 구현 핵심
- 대상 실행 경로
  - `src/run_leaf_rank.py`
  - `src/bash/baselines/run_baseline3_leaf_only_loop_emr.sh`
- 원본 launcher인 `src/bash/baselines/run_baseline3_leaf_only_loop.sh`는 수정하지 않고 유지.
- 새 인자
  - `--leaf_emr_memory_mode off|accumulated`
  - `--leaf_emr_history_rank_topk`
  - `--leaf_emr_doc_topk`
  - `--leaf_emr_sent_topk`
  - `--leaf_emr_compression on|off`
  - `--leaf_emr_memory_max_tokens`
- 새 prompt
  - `agent_executor_v1_icl2_emr_memory`
  - 기존 `agent_executor_v1_icl2`를 유지하고, memory-aware variant를 분리함.

## Paper-Faithful 기준
- `History of Recent Actions`는 누적된다.
- `Memory of Documents`도 누적 document pool을 기반으로 만든다.
- document compression은 per-document top-k가 아니라, 누적 pool 전체 문장에 대해 query-conditioned global filtering을 수행한다.
- 현재 구현은 논문 문구에 맞춰 global top-k sentence selection을 사용하며, 기본값은 `k=10`이다.

## 기본값의 출처
- `--leaf_emr_sent_topk=10`
  - EMR 논문 본문의 global filtering 설명에 맞춘 값이다.
  - 의미는 "문서별 sentence top-k"가 아니라, cumulative document pool 전체에서 globally selected sentence top-k이다.
- `--leaf_emr_compression=on`
  - paper-faithful 기본값이다.
  - `on`이면 cumulative document pool 위에서 global sentence filtering을 적용한다.
  - `off`는 ablation 용도로, full document text를 그대로 memory에 넣고 token budget만 적용한다.
- `--leaf_emr_memory_max_tokens=0`
  - `0`은 auto 모드다.
  - rewrite prompt 전체를 실제 LLM tokenizer로 토크나이즈하고, model context window 안에 들어오도록 document memory를 자동 축소한다.
  - 양수 값을 직접 주면 document memory block token 수의 상한으로 동작한다.

## Mode 해석
- `accumulated`
  - paper-faithful main setting.
  - 이전 iteration들에서 본 문서를 deduplicated bank로 유지한다.
  - `leaf_emr_compression=on`이면 rewrite 시에는 이 누적 pool 전체 문장들을 query-conditioned로 global scoring하고, top-k sentences만 남긴다.
  - top-k에 기여한 문서만 `Memory of Documents`에 남으므로 irrelevant documents는 문장 단계에서 암묵적으로 prune된다.
  - `leaf_emr_compression=off`이면 누적 pool의 full document text를 그대로 memory에 넣고, token budget만 적용한다.

## Baseline 실행
- 메모리 없는 기존 baseline은 그대로 `src/bash/baselines/run_baseline3_leaf_only_loop.sh`로 실행한다.
- 같은 EMR launcher 표면에서 memory만 끄고 싶으면 `LEAF_EMR_MEMORY_MODE=off bash src/bash/baselines/run_baseline3_leaf_only_loop_emr.sh`를 사용하면 된다.

## Memory 구조
### 1. History of Recent Actions
- EMR 스타일을 따라 plain text log로 저장.
- 각 항목은 다음 정보를 포함.
  - `Action`
  - `Query`
  - `Ranks` = iteration별 top retrieved `doc_id` 목록
- 현재 step은 rewrite가 끝난 뒤에 append되므로, 다음 prompt부터 보이게 구현.

### 2. Memory of Documents
- 누적 document pool에서 시작한다.
- 각 문서를 sentence split한 뒤, 모든 문장을 하나의 글로벌 후보 집합으로 합친다.
- MiniLM cross-encoder (`cross-encoder/ms-marco-MiniLM-L6-v2`)로 query-conditioned sentence scoring을 수행한다.
- top-k overall sentences만 남긴다.
- 선택된 sentence를 가진 문서들만 `Memory of Documents`에 나타난다.
- sentence split은 `spacy`가 있으면 sentencizer를 사용하고, 없으면 regex fallback 사용.

## 주의점
- 이번 구현은 prompt-only memory 실험이다. retrieval metric, retrieval order, evaluation 방식은 바꾸지 않았다.
- 논문 재현 관점에서는 `accumulated + compression=on + global sentence filtering(top-k=10)`이 중심이다.
