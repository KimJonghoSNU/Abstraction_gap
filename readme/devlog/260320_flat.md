# 2026-03-20 Round6 flat-descendant ended-reseat

## 목적

`round6`의 현재 ended-reseat는 active beam 쪽에서 `direct child`만 다음 branch 후보로 본다. 이번 구현의 목적은 이 local continuation을 `descendant` 단위까지 넓혀서,

- `run_round3`의 flat branch retrieval 감각을
- `round6`의 tree-constrained retrieval pool 위에 얹는 것

이다.

핵심 질문은 두 개다.

- active branch를 `child-only`가 아니라 `descendant-flat`으로 고르면 더 좋은 중간 branch state를 찾는가?
- leaf를 만나 ended slot이 생겼을 때, replacement candidate를 `whole-tree flat`으로 둘지 `go-explore direct child`로 둘지 어떤 쪽이 더 나은가?

## 구현

### 1. Active-slot controller

새 option:

- `--round6_expandable_candidate_mode {direct_children, descendant_flat}`

이번 실험에서 기본은 `descendant_flat`이다.

의미:

- iter 0:
    - 전체 non-root non-leaf branch를 candidate로 둔다.
- iter 1 이후:
    - `selected_before`의 `strict descendant non-leaf branch` 전체를 candidate로 둔다.
- candidate scoring:
    - 현재 `round5_selector_mode`를 그대로 재사용한다.
    - 기본 실험값은 `meanscore_global`이다.
- selection:
    - slot quota 없이 `global union top-B`

즉, active slot은 더 이상 `direct child only`가 아니라, 현재 선택된 branch subtree 안의 더 깊은 branch로 바로 jump할 수 있다.

### 2. Ended-slot replacement

새 option:

- `--round6_expandable_ended_scope {leftover_expandable, whole_tree_flat, goexplore_direct_child}`

이번 실험에서 보는 두 새 scope:

- `whole_tree_flat`
    - semantic tree 전체 non-leaf branch를 candidate로 둔다.
    - flat retrieval 성향이 가장 강하다.
- `goexplore_direct_child`
    - root와 현재 선택된 branch들의 decision point에서, 아직 exploit되지 않은 `direct child`만 candidate로 둔다.
    - Go-Explore 해석에 더 가깝다.

기존 `leftover_expandable`은 baseline 호환용으로 남겨뒀다.

### 3. Materialize

descendant-flat / whole-tree-flat 쪽은 retrieval이 아직 prediction tree에 instantiate되지 않은 deeper branch를 바로 고를 수 있다.

그래서 선택된 branch path를 다음 iter traversal이 실제 beam state로 쓸 수 있게, `seed_beam_from_gate_paths(..., reset_history=False)`로 prediction tree 안에 materialize하도록 바꿨다.

의미:

- retrieval이 고른 deeper branch path를
- 다음 iter child-slate 생성이 가능한 실제 `beam_state_paths`로 바꾼다.

### 4. 유지한 것

이번 변경에서 일부러 안 바꾼 것:

- rewrite prompt
- rewrite evidence construction
- `query_pre`, `query_post`
- eval retrieval
- leaf를 만났을 때 ended slot만 replacement한다는 큰 semantics

즉 이번 실험은 rewrite ablation이 아니라 controller / branch-transition ablation이다.

## 실행

기존 `src/bash/round5/run_round6_expandable.sh`는 유지하고, 이번 flat-descendant 실험은 별도 launcher로 분리했다.

실험 launcher:

```bash
bash src/bash/round5/run_round6_flat.sh
```

현재 launcher 기본값:

- `ROUND6_EXPANDABLE_MODE=ended_reseat`
- `ROUND6_EXPANDABLE_CANDIDATE_MODE=descendant_flat`
- `ROUND6_EXPANDABLE_ENDED_SCOPES="whole_tree_flat goexplore_direct_child"`
- `ROUND5_SELECTOR_MODES=meanscore_global`
- `MAX_BEAM_SIZE=10`

결과 suffix 예시:

- `..._expandable_ended_reseat_descendant_flat_whole_tree_flat`
- `..._expandable_ended_reseat_descendant_flat_goexplore_direct_child`

## 검증

정적 검증:

```bash
python -m py_compile src/run_round6.py src/hyperparams.py src/tree_objects.py
bash -n src/bash/round5/run_round6_flat.sh
```

둘 다 통과.

Smoke run:

- `biology`, `num_eval_samples=1`, `num_iters=2`로 코드 path를 확인하려고 했지만
- 현재 retriever model loading 단계에서 CUDA OOM이 나서 full runtime smoke는 아직 못 끝냈다.
- 즉, 이번 커밋은 정적 검증까지는 끝났고, full 실험 결과는 아직 append 전이다.

## 결과 메모

실험 후 아래만 비교해서 append하면 된다.

- baseline `ended_reseat` (`direct_children + leftover_expandable`)
- 새 `descendant_flat + whole_tree_flat`
- 새 `descendant_flat + goexplore_direct_child`

우선 볼 값:

- `ndcg_end`
- `ndcg_max`
- `ndcg_max_iter`

읽는 기준:

- `whole_tree_flat`이 더 좋으면
    - flat retrieval식 teleport가 실제로 local trap 탈출에 도움을 준다.
- `goexplore_direct_child`가 더 좋으면
    - return-to-decision-point 스타일 explore가 더 안정적이다.
- 둘 다 기존보다 나쁘면
    - 문제는 child-only limitation보다도 rewrite/evidence alignment 쪽일 가능성이 크다.
